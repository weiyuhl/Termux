# Termux 项目依赖配置说明

## 依赖下载链接配置位置

### 1. 根目录 build.gradle
**文件路径**: `build.gradle`

**配置内容**:
```groovy
allprojects {
    repositories {
        google()                                    // Google Maven 仓库
        mavenCentral()                              // Maven 中央仓库
        maven { url uri("$rootDir/本地Maven仓库") }  // 本地 Maven 仓库（优先）
        maven { url "https://jitpack.io" }          // JitPack 仓库（备用）
    }
}
```

**说明**:
- `google()` - 用于下载 Android 官方库
- `mavenCentral()` - 用于下载通用 Java/Android 库
- `本地Maven仓库` - 本地依赖，优先使用，无需网络
- `https://jitpack.io` - termux-am-library 的原始下载源

### 2. termux-shared/build.gradle
**文件路径**: `termux-shared/build.gradle`

**第 31 行**:
```groovy
implementation "com.termux:termux-am-library:v2.0.0"
```

这是唯一需要从 JitPack 下载的 Termux 专有依赖。

## 本地仓库配置

### 目录结构
```
本地Maven仓库/
└── com/
    └── termux/
        └── termux-am-library/
            └── v2.0.0/
                ├── termux-am-library-v2.0.0.aar  (16,687 字节)
                └── termux-am-library-v2.0.0.pom  (1,220 字节)
```

### 工作原理
1. Gradle 按照 repositories 列表顺序查找依赖
2. 首先检查本地 Maven 仓库 (`本地Maven仓库/`)
3. 如果本地找到，直接使用，无需网络
4. 如果本地没有，才会尝试从 JitPack 下载

### 优势
- ✅ 离线构建能力
- ✅ 更快的构建速度
- ✅ 不受外部服务影响
- ✅ 可纳入版本控制

```properties
systemProp.http.proxyHost=127.0.0.1
systemProp.http.proxyPort=10808
systemProp.https.proxyHost=127.0.0.1
systemProp.https.proxyPort=10808
```

当前配置的代理端口: **10808**

## 备份位置

### 构建依赖备份
**路径**: `构建依赖备份/`

包含从 Gradle 缓存备份的原始依赖文件，保持了 Gradle 的缓存目录结构。

### 本地 Maven 仓库
**路径**: `本地Maven仓库/`

按照标准 Maven 仓库结构组织的依赖文件，可直接被 Gradle 使用。

## 故障排除

### 如果构建失败找不到依赖

1. **检查本地仓库文件是否存在**:
   ```bash
   ls 本地Maven仓库/com/termux/termux-am-library/v2.0.0/
   ```

2. **清理 Gradle 缓存后重试**:
   ```bash
   ./gradlew clean --refresh-dependencies
   ```

3. **从备份恢复到 Gradle 缓存**:
   ```bash
   Copy-Item -Path "构建依赖备份/termux-am-library" -Destination "$env:USERPROFILE\.gradle\caches\modules-2\files-2.1\com.termux\" -Recurse -Force
   ```

4. **确保代理配置正确**（如果需要从网络下载）

## 构建命令

### 完整构建流程
```bash
# 1. 复制 bootstrap 文件（如果被 clean 删除）
Copy-Item -Path "Bootstrap文件/bootstrap-*.zip" -Destination "app/src/main/cpp/" -Force

# 2. 构建发布版本
./gradlew assembleRelease

# 3. 查看生成的 APK
ls app/build/outputs/apk/release/
```

### 输出文件
- **文件名**: `termux-app_apt-android-7-release_universal.apk`
- **位置**: `app/build/outputs/apk/release/`
- **大小**: 约 114 MB

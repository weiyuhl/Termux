## 构建流程

### 完整构建命令
```bash
./gradlew assembleRelease
```

### 构建过程
```
1. Gradle 初始化
   └─ 读取 build.gradle 和 gradle.properties

2. 解析依赖
   ├─ 检查本地 Maven 仓库 (本地Maven仓库/)
   │   ├─ 找到 termux-am-library v2.0.0 ✅
   │   └─ 使用本地文件，无需网络下载
   └─ 其他依赖从 Google/Maven Central 下载（已缓存）

3. 准备 Bootstrap 文件
   ├─ 执行 :app:downloadBootstraps 任务
   ├─ 检查 app/src/main/cpp/bootstrap-*.zip
   │   ├─ 文件存在且校验和正确 ✅
   │   └─ 跳过复制
   └─ 输出: "Bootstrap file already exists with correct checksum"

4. 编译原生代码
   ├─ NDK 编译 C/C++ 代码
   ├─ 使用 .incbin 指令嵌入 bootstrap zip 文件
   └─ 生成 libtermux-bootstrap.so

5. 编译 Java 代码
   ├─ 编译 terminal-emulator 模块
   ├─ 编译 terminal-view 模块
   ├─ 编译 termux-shared 模块
   └─ 编译 app 模块

6. 打包 APK
   ├─ 合并资源
   ├─ 混淆代码（ProGuard）
   ├─ 打包 DEX 文件
   ├─ 打包原生库
   └─ 生成 APK

7. 输出
   └─ app/build/outputs/apk/release/
       └─ termux-app_apt-android-7-release_universal.apk
```

## 故障排除

### 问题 1: 构建失败，找不到依赖
**解决方法**:
1. 检查 `本地Maven仓库/` 目录是否存在
2. 检查文件是否完整
3. 从 `构建依赖备份/` 恢复

### 问题 2: 找不到 Bootstrap 文件
**解决方法**:
1. 检查 `Bootstrap文件/` 目录
2. 运行 `./gradlew :app:downloadBootstraps`
3. 检查 `app/src/main/cpp/` 目录

### 问题 3: 校验和不匹配
**解决方法**:
1. 删除有问题的文件
2. 从备份恢复或重新下载
3. 验证文件完整性

### 问题 4: 网络下载失败
**解决方法**:
1. 检查代理配置
2. 检查网络连接
